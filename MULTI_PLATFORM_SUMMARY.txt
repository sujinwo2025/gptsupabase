UBUNTU/DEBIAN AUTO-DETECTION
─────────────────────────────────────────────────────────────

What it does:
  1. Detects OS name (ubuntu, debian, etc.)
  2. Detects OS version
  3. Checks for required dependencies:
     - openssl
     - docker
     - docker-compose
  4. Provides platform-specific install instructions if missing
  5. Exits gracefully with helpful error messages

Example output:
  "OS: ubuntu 22.04"
  "✓ All dependencies found"

  OR if missing:
  "✗ Missing dependencies: docker, docker-compose"
  "Installation instructions:"
  "  sudo apt-get update"
  "  sudo apt-get install -y openssl docker.io docker-compose"

Code location: init.sh lines 5-45


VERCEL SERVERLESS DEPLOYMENT
─────────────────────────────────────────────────────────────

Architecture:
  - Git repository → GitHub
  - Push triggers automatic deployment
  - Vercel builds and deploys api/index.js
  - Express app runs as serverless function
  - Routes handled via vercel.json

Environment Variables:
  - Set in Vercel Dashboard
  - Auto-injected into functions
  - Same as Docker Compose setup

Files involved:
  - vercel.json (routing & config)
  - api/index.js (serverless handler)
  - package.json (build scripts)

Deployment methods:
  1. Git-based (recommended)
     - Push to GitHub
     - Auto-deploy via webhook
  2. CLI-based
     - vercel --prod
     - Add env vars: vercel env add VAR_NAME
  3. GitHub Actions
     - Auto-deploy on push
     - Full CI/CD integration

Example: vercel.json
  - Specifies api/** as serverless functions
  - Routes all traffic to api/index.js
  - Memory: 1024MB per function
  - Timeout: 60 seconds

Cost comparison:
  - FREE: 10 requests/sec, 2,000 builds/month, 100GB bandwidth
  - PRO: $20/month for unlimited scaling


DEPENDENCY AUTO-CHECKING
─────────────────────────────────────────────────────────────

What it checks:
  1. openssl (for certificate generation)
  2. docker (for containerization)
  3. docker-compose (for orchestration)

Where it checks:
  - /usr/bin/
  - /usr/local/bin/
  - PATH environment variable

Installation prompts:
  - Ubuntu/Debian: apt-get install
  - CentOS/RHEL: yum install
  - Alpine: apk add
  - Generic: generic installation link

Exit behavior:
  - If all found: continues
  - If any missing: shows instructions and exits
  - User can then install and retry

Code location: init.sh lines 40-75
